# Gaming Assets

This folder contains all media assets for the gaming portfolio.

## Folder Structure

### `/images`
- Game development screenshots
- Unity/Unreal project visuals
- Game jam submissions
- Streaming setup photos
- QA testing documentation

**Recommended formats:** JPG, PNG, WebP
**Naming convention:** `project-name-description.jpg`

Examples:
- `procedural-rpg-gameplay.jpg`
- `vr-physics-playground-screenshot.png`
- `game-jam-2024-submission.jpg`
- `unity-editor-setup.png`

### `/videos`
- Gameplay recordings
- Development timelapse videos
- Tutorial content
- Stream highlights
- Game jam entries

**Recommended formats:** MP4 (H.264), WebM
**Naming convention:** `project-name-type.mp4`

Examples:
- `procedural-rpg-demo.mp4`
- `vr-playground-gameplay.mp4`
- `unity-tutorial-series.mp4`
- `game-jam-timelapse.mp4`

### `/thumbnails`
- Game project thumbnails
- Stream thumbnails
- Tutorial preview images
- Game jam submission cards

**Recommended size:** 1280x720px (16:9 ratio)
**Naming convention:** `project-name-thumb.jpg`

Examples:
- `procedural-rpg-thumb.jpg`
- `vr-playground-thumb.jpg`
- `unity-tutorial-thumb.jpg`

### `/docs`
- Game design documents
- QA test reports
- Game jam postmortems
- Development logs
- Technical documentation

**Recommended formats:** PDF, MD
**Naming convention:** `project-name-document-type.pdf`

Examples:
- `procedural-rpg-design-doc.pdf`
- `vr-playground-postmortem.pdf`
- `qa-testing-methodology.pdf`

## External Links

For Twitch streams, YouTube videos, or itch.io games, reference them in `gaming-data.js` using URLs.

## Performance Tips

1. Compress gameplay footage (use Handbrake or similar)
2. Use WebP for screenshots
3. Keep GIFs under 5MB (or convert to video)
4. Optimize game build screenshots
5. Use lazy loading for all media
